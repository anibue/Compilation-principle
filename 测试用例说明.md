# 测试用例说明

本文档说明 `test_cases/` 目录中测试用例的分组、验证目标和推荐使用方式。测试文件包括正向执行用例、错误处理用例和边界用例。

## 用例分类

| 分类 | 用例 | 验证目标 |
| --- | --- | --- |
| 基础功能 | `E01`, `E0101`, `P9102` | 变量声明、赋值、表达式和 `WRITE` |
| 原 PL/0 功能 | `P9101`, `P9104`, `T1` | 过程调用、嵌套过程、`WHILE` 和作用域 |
| 条件分支 | `E110`, `ELSE_NOTEQ` | `IF ... THEN ... ELSE`、`<>`、`!=` |
| FOR 循环 | `FORT01`, `FORT02`, `FORT03` | `FOR TO`、`FOR DOWNTO`、复合语句循环体 |
| 复合赋值 | `COMP01` | `+=`, `-=`, `*=`, `/=` |
| 自增自减 | `INC01` | `++`, `--` |
| 提前返回 | `RET01` | 过程中的 `RETURN` |
| 错误处理 | `EX000`, `EX01`, `EX10`, `EX11`, `T2`, `test1` | 非法位置 token、读入语法错误、重复标识符等 |
| 边界用例 | `EDGE01`-`EDGE11` | 循环边界、嵌套循环、表达式边界、单项扩展语法 |

文件扩展名为 `.PL0` 的是源程序，`.COD` 是编译或运行后的参考输出。部分边界用例可能只有源程序，需根据当前实现重新生成输出。

## 代表用例

| 用例 | 重点 | 期望现象 |
| --- | --- | --- |
| `E110.PL0` | `ELSE` 和 `<>` | 输入不同值时进入不同分支 |
| `ELSE_NOTEQ.PL0` | `ELSE` 和 `!=` | 条件为假时执行 `ELSE` 分支 |
| `COMP01.PL0` | 四类复合赋值 | 依次输出 `15`, `10`, `50`, `10` |
| `INC01.PL0` | 自增自减 | 验证变量加一和减一 |
| `FORT01.PL0` | 正向 FOR 循环 | 输出 `1+2+3+4+5` 的结果 `15` |
| `FORT02.PL0` | 反向 FOR 循环 | 输出 `5+4+3+2+1` 的结果 `15` |
| `FORT03.PL0` | 复合循环体 | 输出循环变量并最终输出 `55` |
| `RET01.PL0` | 提前返回 | `RETURN` 后的语句不再执行，输出 `100` |

## 边界用例

| 用例 | 验证点 |
| --- | --- |
| `EDGE01.PL0` | `FOR` 起止值相同 |
| `EDGE02.PL0` | 嵌套 `FOR` 循环 |
| `EDGE03.PL0` | `FOR` 边界使用表达式 |
| `EDGE04.PL0` | 复合赋值右侧为表达式 |
| `EDGE05.PL0` | `IF ... ELSE` 与 `FOR` 嵌套 |
| `EDGE06.PL0` | 空范围 `FOR` |
| `EDGE07.PL0` | 循环体内修改循环变量 |
| `EDGE08.PL0` | 过程调用和乘法运算 |
| `EDGE09.PL0` | 单独验证 `-=` |
| `EDGE10.PL0` | 单独验证 `++` |
| `EDGE11.PL0` | 单独验证 `--` |

边界用例用于观察实现细节，不一定全部纳入自动回归脚本。写实验报告时，应说明它们验证的是边界行为还是错误处理行为。

## 错误处理用例

| 用例 | 说明 |
| --- | --- |
| `EX000.PL0` | 将扩展 token 放在非法语法位置，验证错误报告 |
| `EX01.PL0` | 验证扩展 token 的词法识别和语法恢复 |
| `EX10.PL0` | 验证 `READ` 相关错误处理 |
| `EX11.PL0` | 验证 `!=` 与 `READ` 组合场景下的错误处理 |
| `T2.PL0` | 验证重复标识符等异常情况 |
| `test1.pl0` | 覆盖符号、保留字和非法字符识别 |

这些用例不以“成功执行”为目标。它们的价值在于确认编译器能识别非法输入，并给出可定位的错误信息。

## 运行方式

### GUI 运行

1. 启动 `project/PL01.exe`。
2. 点击 `Refresh` 扫描测试用例。
3. 在下拉框中选择 `.PL0` 文件。
4. 点击 `RUN` 查看编译和执行结果。
5. 需要重新运行时，可先点击 `Clear` 清理输出区域。

### 控制台运行

```powershell
cmd /c test_console\pl0_test.exe test_cases\COMP01.PL0 build_tmp\COMP01.COD
Get-Content build_tmp\COMP01.COD
```

控制台输出中，`~~~ RUN PL0 ~~~` 和 `~~~ END PL0 ~~~` 之间是程序运行结果。

### 自动回归

```bash
cd test_console
bash run_tests.sh
```

`run_tests.sh` 主要覆盖正向执行路径。对于错误处理和边界行为，应结合单个用例输出进行人工检查。
